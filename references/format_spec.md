# 标准公文格式 · 完整规格

本文件记录国企体制内常用的标准化公文模板排版规格。
整体遵循《党政机关公文格式 GB/T 9704-2012》，标题字号采用集团惯用的小二号。

## 一、字体与字号规格

| 段落角色 | 识别特征 | 字体（中文） | 字号 | 对齐 | 缩进 |
|---------|---------|------|------|------|------|
| 标题 / 单位抬头 | 文首 1～3 行，含「关于……的报告/汇报/通知」等 | 方正小标宋简体 | 小二号（18pt） | 居中 | 无 |
| 称谓 | 标题后第一行，短句且以冒号结尾，如「各位领导：」 | 仿宋_GB2312 | 三号（16pt） | 左对齐 | 顶格 |
| 正文 | 普通段落 | 仿宋_GB2312 | 三号（16pt） | 两端对齐 | 首行缩进 2 字符 |
| 一级标题 | 以「一、二、……」开头 | 黑体 | 三号（16pt） | 两端对齐 | 首行缩进 2 字符 |
| 二级标题 | 以「（一）（二）……」开头 | 楷体_GB2312 | 三号（16pt） | 两端对齐 | 首行缩进 2 字符 |
| 三级序号 | 以「1. 2. ……」开头 | 仿宋_GB2312 | 三号（16pt） | 两端对齐 | 首行缩进 2 字符 |
| 四级序号 | 以「（1）（2）……」开头 | 仿宋_GB2312 | 三号（16pt） | 两端对齐 | 首行缩进 2 字符 |
| 附件说明 | 以「附件」开头 | 仿宋_GB2312 | 三号（16pt） | 左对齐 | 顶格 |
| 落款（单位、日期） | 文末单位名与「××××年×月×日」 | 仿宋_GB2312 | 三号（16pt） | 右对齐 | 无 |

要点：三级、四级序号与正文的字体字号完全一致，仅靠序号区分，因此脚本把它们统一按「正文」处理。
一级、二级标题与正文唯一的区别是字体（黑体 / 楷体），字号、缩进、行距均与正文相同——它们是带序号的段落，不是居中独立大标题。

**西文字体**：上表「字体」列只作用于中文字符。所有**数字与英文一律用 Times New Roman**——
脚本对每个 run 分别设置东亚字体（eastAsia，管中文）与西文字体（ascii/hAnsi/cs = Times New Roman，
管数字和拉丁字母），因此「2026年」「用 Python 写」这类中英混排会由 Word 自动分流显示。
若要换西文字体，改常量 `FONT_WESTERN`。

## 二、段落与页面规格

- **行距**：固定值 28 磅（全文统一，含标题与空行）
- **段前**：一级标题段前 1 行、二级标题段前 0.5 行，其余段落 0
  （写入 beforeLines，Word/WPS 段落设置里显示为「N 行」；并按 28 磅行距折算磅值兜底）
- **段后**：大标题段后 1.5 行；其余段落 0
- **表格段前**：每个表格整体段前 0.5 行——技术上落在**表格上方相邻段落的段后**上
  （OOXML 表格自身无段前属性）。表格若在文首或紧接另一表格（上方无段落）则跳过；
  该实现天然幂等，重复套用不会累加、也不会插入空段落
- **正文首行缩进**：2 字符
- **纸张**：A4（21cm × 29.7cm）
- **页边距**：上 3.7cm、下 3.5cm、左 2.8cm、右 2.6cm
- **文字颜色**：全文黑色（套格式时会把彩色文字归一为黑色，并清除多余加粗）

## 三、编号、标点与空格规范化（会改写正文内容）

除字体格式外，脚本默认还纠正四类 AI 常见的不合规写法：

### 3.1 二级编号：数字分级 → 公文括号序号

AI 爱把二级标题写成「1.1 / 1.2 / 2.1」，公文标准是「（一）（二）」。脚本把**识别为二级标题**
的「N.M」按小数点后的数字 M 转成「（M 中文）」，因此每个一级标题下会自动从「（一）」重新计数：

| 原编号 | 转换后 | 原编号 | 转换后 |
|--------|--------|--------|--------|
| 1.1 | （一） | 2.1 | （一） |
| 1.2 | （二） | 2.2 | （二） |
| 1.3 | （三） | 3.1 | （一） |

安全边界：
- 只转**编号后紧跟空格/顿号/行尾**的（如「1.1 基本情况」），不会误伤正文中的「1.1万元」「增长1.5倍」；
- 「1.」这类**单级**序号是公文三级，保持仿宋正文，不转；
- **一级标题不自动把阿拉伯数字转中文**——公文里「1.」本身是三级序号，与数字大标题无法可靠区分，
  为避免误伤正文，一级「1.」需人工确认后改成「一、」。原文一级已是「一、」则不受影响。

关掉此项：`--keep-numbering`。

### 3.2 三级优先：不许跳级用「（1）」

公文层级顺序为 一、→（一）→ 1. →（1）。若某节（上一个一/二级标题之后）尚未出现
三级序号「1.」，却直接出现四级式「（1）（2）」，脚本把该节这些段落提升为三级「1. 2.」
（保留原数字，逐节独立判断）。该节已先出现三级序号时，其后的「（N）」视为正常四级，
保持不变。关掉此项：`--keep-numbering`（与 3.1 共用开关）。

### 3.3 英文双引号 → 中文双引号

全文（含表格单元格）把英文直双引号 `"` 按段内成对次序转成中文「“ ”」。
正常成对的引号方向正确；若原文引号不成对，`--report-only` 打印的待转数量可帮助发现异常。
关掉此项：`--keep-quotes`。

（单引号 `'` 因存在英文撇号 it's 之类歧义，脚本不自动转，需要时手动处理。）

### 3.4 数字/英文两侧空格清理

AI 生成的文稿爱在数字、英文两侧加空格（「共 23 条」「用 Python 写」），公文体例不留。
全文（含表格单元格）按以下规则清理：

| 情形 | 处理 | 例子 |
|------|------|------|
| 任一侧是中文/全角字符 | 删除 | 共 23 条 → 共23条；用 Python 写 → 用Python写 |
| 两侧都是字母/数字 | 保留 | Claude Code、GB/T 9704 |
| 数字挨着半角符号 | 删除 | 2514 / 点赞 → 2514/点赞 |
| 纯英文语境的标点 | 保留 | e.g. test |

段首用于对齐的缩进空格（如附件续行）不受影响；跨 run（不同格式片段）的空格也能正确处理。
关掉此项：`--keep-spaces`。

## 五、如何调整格式规格

所有规格都集中在脚本 `scripts/apply_gongwen_format.py` 顶部的「格式规格」区块，是一组常量。
需要改格式时，改这些常量即可，无需改动脚本逻辑：

| 常量 | 含义 | 当前值 |
|------|------|--------|
| `FONT_TITLE` | 标题字体 | 方正小标宋简体 |
| `FONT_BODY` | 正文字体 | 仿宋_GB2312 |
| `FONT_H1` | 一级标题字体 | 黑体 |
| `FONT_H2` | 二级标题字体 | 楷体_GB2312 |
| `SIZE_TITLE` | 标题字号（磅） | 18（小二） |
| `SIZE_BODY` | 正文字号（磅） | 16（三号） |
| `FONT_WESTERN` | 数字/英文（西文）字体 | Times New Roman |
| `LINE_SPACING_PT` | 行距（磅） | 28 |
| `TITLE_AFTER_LINES` | 大标题段后（行） | 1.5 |
| `H1_BEFORE_LINES` | 一级标题段前（行） | 1.0 |
| `H2_BEFORE_LINES` | 二级标题段前（行） | 0.5 |
| `TABLE_BEFORE_LINES` | 表格整体段前（行） | 0.5 |
| `FIRST_LINE_CHARS` | 正文首行缩进（字符） | 2 |
| `MARGIN_*_CM` | 上下左右页边距（厘米） | 3.7 / 3.5 / 2.8 / 2.6 |

常见调整示例：
- 标题改回公文标准的「二号」→ 把 `SIZE_TITLE` 改为 22。
- 行距改成「1.5 倍」或「30 磅」→ 改 `LINE_SPACING_PT`（脚本用的是固定值行距）。
- 字体名称在不同电脑上不一致 → 例如把 `仿宋_GB2312` 换成 `仿宋`、`楷体_GB2312` 换成 `楷体`。

## 六、字体安装说明

四个公文标准字体：`仿宋_GB2312`（正文主体）、`方正小标宋简体`（大标题）、
`楷体_GB2312`（二级标题）、`黑体`（一级标题，系统自带无需装）。三个需装字体的内部名称与
脚本写入 docx 的名称完全一致，装上即生效。

字体包不放代码仓（体积较大，且含方正等商业字体，不宜公开分发），由内部渠道分发
（企业云盘/知识库等，或向管理员获取）；`fonts/安装说明.md` 提供 Mac / Windows 安装步骤与按需安装建议。
WPS Office 自带这些字体，WPS 用户无需安装。文档里的字体名称本身始终合规——
即便本机没装、预览被替换，发给装了字体的同事或用 WPS 打开即显示正常。

# 标准公文格式

把一份已经写好内容的 Word 文档，一键排版成国企体制内通用的标准化公文格式。
适用于工作汇报、报告、通知、请示、意见、总结等公文类文档。

## 格式规格速查

| 段落 | 字体（中文） | 字号 | 对齐 / 缩进 / 间距 |
|------|------|------|------|
| 标题 / 单位抬头 | 方正小标宋简体 | 小二号 18pt | 居中，段后 1.5 行 |
| 正文 | 仿宋_GB2312 | 三号 16pt | 两端对齐，首行缩进 2 字符 |
| 一级标题「一、」 | 黑体 | 三号 16pt | 两端对齐，首行缩进 2 字符，段前 1 行 |
| 二级标题「（一）」 | 楷体_GB2312 | 三号 16pt | 两端对齐，首行缩进 2 字符，段前 0.5 行 |
| 三级「1.」/ 四级「（1）」 | 仿宋_GB2312 | 三号 16pt | 同正文 |
| 附件说明 | 仿宋_GB2312 | 三号 16pt | 顶格 |
| 落款（单位、日期） | 仿宋_GB2312 | 三号 16pt | 右对齐 |

**数字与英文一律用 Times New Roman**（西文字体），中文才用上表的仿宋/黑体/楷体/标宋——
同一段中英混排会自动分流。**表格**整体段前 0.5 行（落在表格上方相邻段落的段后）。
行距固定值 28 磅；标题段后 1.5 行、一级标题段前 1 行、二级标题段前 0.5 行，其余段落段前段后为 0；
A4 纸；页边距上 3.7 / 下 3.5 / 左 2.8 / 右 2.6 cm。
完整规格、字号对照表、调整方法见 `references/format_spec.md`。

## 自动规范化（除字体格式外，脚本还会改四类「内容」）

AI 写的公文常有几类不合规范的写法，脚本默认自动纠正：

1. **二级编号**：AI 爱用数字分级「1.1 / 1.2 / 2.1」，公文标准是「（一）（二）」。脚本把
   识别为二级标题的「N.M」按小数点后的数字转成「（一）（二）…」（每个一级标题下自动重新计数），
   并套楷体。一级标题「一、」保持不变。
   - 只转**紧跟空格/顿号的编号**（如「1.1 基本情况」），不会误伤正文里的「1.1万元」；
   - 「1.」这类单级序号是公文三级，保持仿宋正文，不动。
2. **三级优先**：公文层级为 一、→（一）→ 1. →（1），不允许跳级。若某节还没出现三级序号
   「1.」却直接用了「（1）（2）」，这些段落自动提升为三级「1. 2.」（保留原数字）；
   该节已有三级序号时，「（N）」视为正常四级，保持不变。
3. **英文双引号 → 中文双引号**：全文（含表格）把 `"` 按成对次序转成「“ ”」。
4. **数字/英文两侧空格清理**：AI 爱写「共 23 条」「用 Python 写」，公文体例不留这种空格，
   全文（含表格）自动删除；英文单词间（Claude Code）、字母与数字间（GB/T 9704）的
   必要空格保留。

四项默认开启。想保留原样：`--keep-numbering`（编号两项）、`--keep-quotes`（引号）、
`--keep-spaces`（空格）。`--report-only` 会列出全部转换对照与数量，便于套用前核对。

## 操作流程

排版由脚本 `scripts/apply_gongwen_format.py` 完成。它会自动识别每个段落的角色
（标题、正文、各级标题、附件、落款），再套上对应格式。`.doc` 文件会先自动转换成 `.docx`。

**第 1 步 · 检查依赖**
脚本依赖 python-docx。如未安装，先运行：
```
pip3 install python-docx
```

**第 2 步 · 先预览段落识别结果**
带 `--report-only` 跑一遍，只打印分类、不生成文件：
```
python3 ~/.claude/skills/gongwen-format/scripts/apply_gongwen_format.py "文档路径" --report-only
```
对照原文核对这份分类报告，重点看两件事：
- **标题区行数**对不对（单位抬头 + 标题通常 1～2 行）；
- **一级 / 二级标题**有没有认错（靠「一、」「（一）」开头识别）。

**第 3 步 · 如有识别错误，做针对性修正**
- 标题区行数不对 → 加 `--title-lines N`（强制把开头 N 个非空段落作为标题区）。
- 个别段落角色错了 → 先正常生成，再用 python-docx 单独把那几段改成正确字体即可。

**第 4 步 · 正式生成**
去掉 `--report-only`：
```
python3 ~/.claude/skills/gongwen-format/scripts/apply_gongwen_format.py "文档路径"
```
输出文件默认生成在原文件旁边，命名为「原文件名（公文格式）.docx」。原文件不会被改动。
可用 `-o "输出路径"` 自定义。

**第 5 步 · 向用户交代结果**
告诉用户输出文件路径，并简要说明套用了哪些格式，以及下方「注意事项」中与本次文档相关的点。
若用户或收件人电脑可能没装公文字体，提示 `fonts/` 目录有字体包和《安装说明.md》，可按需安装
（黑体系统自带无需装；用 WPS 打开则免安装）。

## 注意事项

- **彩色文字会被统一成黑色、多余加粗会被清除**——这是公文规范的一部分。若用户文中有
  需要保留的彩色或加粗强调，先提醒用户。
- **编号与空格会改写正文内容**（1.1 →（一）、无三级时（1）→ 1.、多余空格删除）。
  套用前建议用 `--report-only` 让用户核对对照表；转换结果也会打印在最后。
- **一级标题不自动把数字转中文**。因为公文里「1.」本身是三级序号，无法与数字大标题可靠区分，
  强转会误伤正文。若原文一级是「1.」式，请让用户确认后手动改成「一、」，或先把一级改好再套。
- **引号按成对次序转换**。正常成对的英文引号会正确转为「“ ”」；若原文引号不成对（漏引号），
  可能出现方向错误，`--report-only` 的待转数量可帮助发现异常。
- **表格**只统一字体为仿宋，不改字号和表格结构（避免撑破单元格）。
- **页眉、页脚、页码**不会自动添加或修改。如用户需要加页码等，单独处理。
- **文本框、图形、艺术字**里的文字脚本无法处理，需手动调整。
- `.doc` 转 `.docx` 依赖 macOS 自带的 `textutil`。非 macOS 环境请让用户先在
  Word/WPS 里「另存为」`.docx`。
- **字体按需安装**：公文标准字体为 `仿宋_GB2312`、`方正小标宋简体`、`楷体_GB2312`
  （`黑体`系统自带无需装）。字体包不放代码仓（体积大且含方正等商业字体，不宜公开分发），
  由内部渠道分发（如企业云盘/知识库，或向管理员获取）；`fonts/安装说明.md` 是 Mac/Windows 安装步骤。
  对方没装时 Word 预览会被替换显示（字体名本身合规），按需装即可——最少装仿宋+标宋，
  有二级标题再加楷体；WPS 自带全部字体，用 WPS 打开免安装。

## 调整格式规格

若用户要修改格式标准本身（如标题改二号、行距改 1.5 倍、换字体名等），
所有规格都是 `scripts/apply_gongwen_format.py` 顶部「格式规格」区块的常量，改常量即可。
对照说明见 `references/format_spec.md` 第四节。
